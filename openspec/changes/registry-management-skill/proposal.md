## Why

The agent surface repograph ships today is read-only: the generated `repograph` skill teaches agents to *consume* the registry (list, status, context, switch) and explicitly forbids mutation â€” "the registry is the user's to manage." That leaves the entire setup path (register a repo, group repos into a workspace, change a repo's metadata) as manual CLI work the agent cannot help with. Agents that can resolve "switch to taverne" should also be able to act on "register this repo and add it to the acme workspace." Closing that gap also exposes two CLI shortfalls that block a safe agent workflow: there is no non-lossy way to *edit* an existing entry, and the mutating commands emit nothing an agent can verify against.

## What Changes

- Add a second generated agent skill, **`repograph-setup`**, that drives the mutating surface (`add`, `remove`, `workspace â€¦`, and the new `edit`) behind a plan â†’ confirm â†’ execute â†’ verify workflow. Generated by `repograph init` from a single canonical source, exactly like the existing read-only skill.
- Introduce a **`Capability`** dimension (Consumer vs Setup) in the artifact layer. Agents that own a whole file (Claude, Cursor) receive a discrete second artifact (`skills/repograph-setup/SKILL.md`, `repograph-setup.mdc`); flat-file agents (AGENTS.md, Aider, Windsurf) get both bodies inlined into their single delimited block.
- Add a new **`repograph edit <name> [--name --description --stack --path]`** command â€” the non-lossy "update existing config" path. `--name` (rename) rewrites workspace membership references so groupings survive the rename.
- Add **`--json` confirmation output** to the mutating commands (`add`, `remove`, `workspace create/add/remove/rm`), which today write nothing to stdout. The agent's verify step becomes one call instead of a follow-up `list --json`.
- **Version-stamp** the managed delimiter marker and add a **read-only** staleness/presence check to `repograph doctor` that flags missing or out-of-date skill artifacts and points the user at `repograph init`. `doctor` stays non-mutating; `init` remains the sole regenerator.
- Update the existing consumer skill body: its "ask the user to run `add` themselves" line now delegates to the `repograph-setup` skill.

Out of scope (v1): driving `repograph init` agent-toolchain reconfiguration from the skill; bulk/recursive directory-scan registration; workspace rename/edit; a `doctor --fix` auto-remediation; auto-rewriting artifacts from the `update` self-update command.

### Modified Capabilities
- `agent-skills`: artifact generation gains a `Capability` dimension; a second `repograph-setup` skill body is emitted (discrete file for wholly-owned agents, inlined block for flat-file agents); the delimiter marker is version-stamped; the consumer body delegates mutation to the setup skill.
- `registry-core`: new `edit` command for non-lossy in-place updates (including rename with workspace-reference rewrite); `add` and `remove` emit structured `--json` confirmation.
- `workspace-support`: `workspace create/add/remove/rm` emit structured `--json` confirmation.
- `doctor-command`: new read-only check reporting missing or stale (version-mismatched) skill artifacts with a `repograph init` remediation hint.

## Impact

- **Core** (`crates/repograph-core/`): `agent_artifact.rs` (Capability dimension, `SETUP_BODY`/`SETUP_SUMMARY`, version-stamped delimiter, `install_artifacts`/`render_artifact`/`resolve_path`/`ArtifactResult` changes); `config.rs` (an `edit_repo` mutator that rewrites workspace member refs on rename).
- **Binary** (`crates/repograph/`): new `commands/edit.rs`; `--json` paths added to `commands/add.rs`, `commands/remove.rs`, `commands/workspace.rs`; doctor freshness check in `commands/doctor.rs`; clap dispatch in `main.rs`.
- **Generated artifacts**: existing `repograph` skill body text changes (delegation line); new `repograph-setup` artifacts land on next `repograph init`. Existing installs surface as "stale" via doctor until re-init.
- **Tests**: `tests/init_artifacts.rs` (two-capability emission, inline vs discrete), new acceptance tests for `edit` and mutator `--json`, doctor staleness test. Real `git2` + `tempdir`, outside-in.
- **Contracts preserved**: exit codes, stdout=data/stderr=diagnostics, delimiter byte-stability outside the managed region, no `unwrap`/`todo!`.
